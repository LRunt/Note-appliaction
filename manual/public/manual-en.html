Introduction

Thank you for choosing Notes to manage your notes and tasks. In this manual, I'll show you how to use the app so you can fully utilize its potential for creating, storing, and organizing your notes.

Launching the App

When the app is launched, an initialization screen with the app's logo appears. This screen indicates that the app is initializing.

Once initialized, the user is presented with the home screen, which is shown in the following image.

On this page, there are two buttons:
  • Three Lines - Opens the slide-out menu
  • Cogwheel - Opens the app settings

Slide-out Menu

The slide-out menu contains a user header, a hierarchical data structure, and a sync button with the last sync date for logged-in users.

Directory

This screen displays a list of files. Files can be handled similarly to a tree structure. At the end of each file, there are three dots that open a menu. Clicking on a file directs the user to that file.

In the bottom right corner, there's a floating button. Clicking on it shows three buttons:

  • Note - Creates a new note in the current directory
  • Folder - Creates a new folder in the current directory
  • Red Cross - Hides the previous buttons

Rich Text Editor

The editor is designed for reading, creating, and editing notes. The application panel shows the path to the note. Below the application panel is the control panel, which can be hidden or shown by a floating button in the bottom right corner of the screen. The following images show the editor's appearance.

Registration

You can access the registration page through the slide-out menu by pressing the registration button. During registration, you can choose to register using an email-password combination or a Google account.

Login

You can access the registration page through the slide-out menu by pressing the login button. The following image shows the login screen with the different parts of this screen labeled.

  1. Return to the previous page
  2. Text field for entering email
  3. Text field for entering password
  4. Link to the password recovery page
  5. Button to log in
  6. Link to the new user registration page
  7. Button to log in with a Google account

Password Recovery

It can happen that a user forgets their password. In this case, there's an option to change the password. The following image shows the password change screen.

This page contains a text field for entering an email address, to which an email with a password change link will be sent. After clicking the Send Email button, an email will be sent to the given address. This email is shown in the next image.

Clicking on the link in the incoming email will redirect the user to a page with a text field and a button to save the new password. This page is shown in the next image.

Settings

The user can access the settings page using the cogwheel on the main page. The following image shows what the settings page looks like.

The following section describes the different settings options

Language

The language option in the settings changes the app's localization. In the current version, two languages are available: Czech and English.

Dark Mode

This option switches the app's appearance mode. The app has a light and a dark mode that can be toggled with this switch in the settings.

Show Conflicts

If a conflict is detected, the local version of the conflicting data is saved. This data can be displayed with the show button next to the Show Conflicts option. The appearance of the conflict page is shown in the next image.

This page displays all conflicts. Each conflict is named after the file and the timestamp when the conflict occurred. Users can view and delete this data.

Delete Local App Data

This option deletes all app data. After pressing it, a dialog will appear to confirm if the user wants to delete all this data. The data is then deleted, and the app returns to its first-open state. This feature was primarily for testing purposes, but it remains for users who want to restore the app to its first-open state.

App Information

Displays a dialog box with information about the app, such as the author, app version, and release date.
